# Technical bug policy

Technical bug tickets include software crashes, errors, upload failures, data synchronization problems, broken features, report generation problems, slow performance, and issues caused by a recent update.

Typical evidence phrases:
- application crashes
- found a bug
- latest update
- report generation
- upload a file
- not syncing data
- performance issue
- error message

Default department: technical_team.

Recommended actions:
1. Identify the affected product and feature.
2. Ask for reproduction steps if they are missing.
3. Check whether the issue is already known.
4. If the ticket mentions a crash, update, report generation, or data synchronization failure, route it to the technical team.
5. Escalate if the issue affects a key workflow, has urgent priority, or appears across multiple customers.

The system must not claim that engineering fixed the bug unless this is present in the ticket or resolution notes.
