# Priority rules for support tickets

This document defines how ticket priority should be interpreted.

Allowed priority values are low, medium, high, and urgent.

Low priority:
The customer reports a minor inconvenience, a general question, or a non-blocking problem. The issue does not prevent the customer from using the main product.

Medium priority:
The issue affects a useful feature or creates noticeable friction, but the customer can still continue working. Medium priority is also used when the issue description is incomplete and the risk is unclear.

High priority:
The issue blocks an important workflow, affects payments, prevents access to an account, breaks a key feature, or may require escalation. High priority requires faster review and clearer operator guidance.

Urgent priority:
The issue is severe, time-sensitive, repeated, connected with security, account access, payment loss, SLA breach, or customer escalation. Urgent tickets must be reviewed quickly and should usually include escalation policy.

Priority must not be increased without evidence. If the text does not support the selected priority, the system must explain uncertainty and keep the priority conservative.
