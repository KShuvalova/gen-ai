# Account access policy

Account access tickets include login failures, password reset problems, missing authentication codes, two-factor authentication delivery issues, account lockouts, and situations where the customer cannot enter the account despite correct credentials.

Typical evidence phrases:
- unable to access my account
- correct credentials
- password reset
- authentication codes are not being delivered
- two-factor authentication
- verification code
- account locked

Default department: support_l2.

Recommended actions:
1. Confirm whether the customer can access the recovery email or phone number.
2. Check recent login attempts and authentication delivery logs.
3. If two-factor codes are not delivered, verify SMS or email provider status.
4. If the customer cannot access the account after correct credentials, reset the account session and initiate secure recovery.
5. Escalate if the ticket is high or urgent, if SLA is breached, or if multiple previous tickets exist.

The response must not invent security events. If the ticket only says that the user cannot log in, do not claim that the account was hacked.
