# Billing and refund policy

Billing and refund tickets include failed payments, duplicate charges, bank account deductions, billing statement discrepancies, invoice issues, refund requests, and payment gateway failures.

Typical evidence phrases:
- payment was deducted
- transaction shows failed
- refund
- recent charge
- billing statement
- duplicate charge
- payment gateway
- invoice

Default department: billing_team.

Recommended actions:
1. Check transaction status in the payment system.
2. Compare the customer's claim with payment gateway records.
3. If money was deducted but the transaction failed, create a payment investigation.
4. If the customer requests a refund, verify eligibility and expected refund timeline.
5. If the issue is high or urgent, or SLA is breached, route through escalation policy.

The system must not invent refund amounts, dates, or bank details. If the ticket does not contain a number or amount, the output must not include one.
