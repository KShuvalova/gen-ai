# Отчёт по домашнему заданию к семинару 5

## Что было реализовано

В рамках домашнего задания был доработан макро-агент на базе ReAct и Tool Use. Основная логика агента сохранена, изменения внесены только в папку `семинар_5/starter`.

Был добавлен шестой инструмент `compare_periods`. Он сравнивает одну макроэкономическую метрику в двух периодах и возвращает значения за оба периода, абсолютное изменение, отношение второго значения к первому и источники данных. Инструмент использует уже существующие функции: `get_fx_rate`, `get_key_rate`, `get_inflation` и `get_unemployment`.

Также была добавлена JSON-схема нового инструмента в `schemas.py`. В системный промпт агента добавлено правило: если пользователь просит сравнить одну и ту же метрику между двумя датами или периодами, агент должен использовать `compare_periods`.

## JSONL-логирование

В `agent.py` добавлено сохранение трассы запуска в файл `trace.jsonl`. В лог записываются вопрос пользователя, итоговый ответ, количество шагов, ошибка при её наличии, trace со всеми вызовами инструментов, usage с токенами и стоимостью, а также сведения о кэше.

Логирование позволяет разбирать, какие инструменты агент вызывал, какие аргументы передавал и на каком шаге получил финальный ответ.

## Расширение eval

Файл `eval.py` был расширен с 4 до 10 тестовых вопросов. Новые кейсы проверяют работу `compare_periods` для курса USD, инфляции и безработицы, сравнение разных макроиндикаторов за один период, расчёт кросс-курса USD/CNY и multi-hop вопрос про ключевую ставку и индекс нищеты.

Для Q2 была добавлена дополнительная логика проверки: агент может корректно решить вопрос либо двумя вызовами `get_fx_rate`, либо одним вызовом `compare_periods`. Так как оба варианта дают правильные данные, `compare_periods` засчитывается как допустимый эквивалент.

Синтаксических ошибок нет. Итоговый результат оценки:

```text

```

Все десять тестовых вопросов прошли проверку.

## Диагностика ошибок

Основная проблема возникла в Q2. Агент недетерминированно выбирал один из двух корректных путей: дважды вызвать `get_fx_rate` или один раз вызвать `compare_periods`. Изначально `eval.py` ожидал только один конкретный инструмент, поэтому корректный ответ мог получать статус FAIL. Проверка была доработана так, чтобы принимать оба валидных варианта поведения.

Ещё одна особенность проявилась в вопросах с текущими данными по инфляции и безработице. Если данных за самый свежий месяц нет, агент делает дополнительные вызовы за предыдущие месяцы и использует последние доступные значения. Это поведение корректно, потому что инструменты возвращают явную ошибку при отсутствии данных, а агент не придумывает числа.
